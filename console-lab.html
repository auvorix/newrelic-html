Console Lab

Each button simulates a real user action — observe raw production-style output in DevTools Console and Network.

Setup:
Open DevTools (F12) → Console tab. Then click any button. The output here is a summary — the full context object is only visible in the DevTools console.
Network tab:
Switch to the Network tab and filter by Fetch/XHR. You will see each API call with its real HTTP status, response headers, and timing.
Where to find in New Relic:
JS Errors — all crash buttons + HTTP 4xx/5xx via noticeError()
AJAX — all fetch buttons with real status + timing
Session Traces — full waterfall of what was triggered

What each section is

Happy path — user succeeds
Client error — user caused it
Server error — backend failed
JS runtime crash — code bug

Happy path — request succeeds

User performs an action and the server responds correctly.

Client errors — user or token caused it

4xx range. The server understood the request but refused or could not fulfil it.

Server errors — backend failed

5xx range. The user did nothing wrong — the server crashed or is unreachable.

JavaScript runtime crashes

No network request. The JavaScript engine itself throws. These appear in New Relic as JS errors with stack traces.
